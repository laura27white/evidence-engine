# Iconography

Icons are semantic, not decorative. Every icon exists because it adds meaning the surrounding text cannot carry alone.

## Rules

- **Pair with a label.** An icon without a text label is only acceptable when the immediate surroundings make its meaning unambiguous (e.g. a trend arrow next to a large number). Otherwise, the icon needs a visible label or a descriptive `aria-label`.
- **Decorative icons are marked `aria-hidden`.** If the text around the icon already conveys the meaning, the icon is decorative and should be hidden from the accessibility tree.
- **No emoji.** The brand voice is editorial, not chatty. Icons are stroked line art from the Lucide catalogue.
- **Stroke and fill.** We use stroked line icons at 1.75px weight. No filled shapes. This ties the icon set visually to the typographic weight of the body font.

## Where icons come from

- **Lucide React** is the default catalogue. Import from `@tp/design-system` rather than `lucide-react` directly so we can swap upstream or apply wrappers in one place.
- **Two domain icons** live in `src/icons/`:
  - `DriftArrowIcon` is a double-stroke arrow used next to drift values. It is visually sturdier at small sizes than Lucide's plain arrow.
  - `CascadeNodeIcon` is a one-to-two glyph used by the Cascade view legend.

## Forbidden patterns

- Colour-only meaning. Every severity colour is paired with an icon and a label.
- Icon-only buttons without `aria-label`.
- Icons used decoratively at the start of a heading "for warmth".
- Custom icons that duplicate a Lucide icon's meaning.
